## Story
**As a** pesquisador de campo usando o app instalado no Android,
**I want** que o backup CSV aconteça automaticamente na pasta Documents do dispositivo a cada registro,
**so that** os dados estejam sempre salvos localmente no Android sem que eu precise selecionar pasta, ativar nada, ou ter acesso a Chrome desktop.

## Acceptance Criteria

1. Quando `Capacitor.isNativePlatform()` retorna `true`, o `addRegistro()` grava o backup via `@capacitor/filesystem` em vez de via File System Access API
2. O arquivo CSV é salvo em `Directory.Documents` com o nome `etograma_backup_{nome-projeto}.csv` — mesmo padrão da Story 1.5
3. O append incremental funciona corretamente: BOM + cabeçalho na primeira linha do arquivo, somente a linha de dados nos registros seguintes
4. O seletor de pasta do modal "Novo Projeto" (Story 1.5) não aparece no contexto nativo — o backup para Documents é silencioso e automático
5. Uma indicação visual discreta ("💾 Documents") aparece na tela home quando o backup nativo está ativo
6. Um botão "Compartilhar CSV" no painel de resumo usa `@capacitor/share` para abrir o share intent do Android — permitindo o usuário enviar o arquivo por WhatsApp, e-mail, Drive, etc.
7. `backupSuportado()` continua retornando `false` no contexto nativo (File System Access API), enquanto uma nova função `backupCapacitorAtivo()` retorna `true` quando `Capacitor.isNativePlatform()` e `@capacitor/filesystem` está disponível
8. Se a escrita falhar (sem permissão, storage cheio), o app exibe toast de aviso mas não interrompe o `addRegistro()` — o registro é salvo normalmente no IndexedDB
9. As permissões necessárias (`READ_EXTERNAL_STORAGE`, `WRITE_EXTERNAL_STORAGE`) já estão declaradas no `AndroidManifest.xml` (Story 1.3 — verificar e complementar se necessário)
10. O comportamento PWA web (Story 1.5) permanece 100% inalterado — as duas implementações coexistem por detecção de contexto

    ```javascript
    async function appendToBackupCSVNativo(r) {
      if (!backupCapacitorAtivo()) return;
      const { Filesystem, Directory, Encoding } = Capacitor.Plugins;
      const path = nomeFichaBackup();
      try {
        // Verificar se arquivo existe para decidir BOM+header
        let vazio = true;
        try {
          const stat = await Filesystem.stat({ path, directory: Directory.Documents });
          vazio = stat.size === 0;
        } catch { vazio = true; }
        const linha = gerarLinhaCsvBackup(r);
        const conteudo = vazio
          ? "﻿" + cabecalhoCSV() + "\n" + linha + "\n"
          : linha + "\n";
        if (vazio) {
          await Filesystem.writeFile({ path, data: conteudo, directory: Directory.Documents, encoding: Encoding.UTF8 });
        } else {
          await Filesystem.appendFile({ path, data: conteudo, directory: Directory.Documents, encoding: Encoding.UTF8 });
        }
      } catch (e) {
        if (e?.message?.includes('permission')) mostrarToast('Sem permissão para backup. Verifique as configurações do app.');
        else mostrarToast('Falha no backup automático.');
      }
    }
    ```

### Directory.Documents no Android

- **Caminho físico:** `/sdcard/Documents/` (Android 10+) ou `/storage/emulated/0/Documents/`
- **Verificação via ADB:** `adb shell ls /sdcard/Documents/etograma_backup_*.csv`
- **Android 9 e abaixo:** requer `WRITE_EXTERNAL_STORAGE` em runtime
- **Android 10+:** sem permissão necessária para Documents via Filesystem plugin (scoped storage)
- **Capacitor `appendFile`:** disponível desde `@capacitor/filesystem` 1.x — usar para incrementalidade

### Coexistência PWA + Nativo

| Contexto | `backupSuportado()` | `backupCapacitorAtivo()` | Backup ativo |

| Firefox desktop | `false` | `false` | Desabilitado |
| APK Android (Capacitor) | `false` | `true` | Via `@capacitor/filesystem` (esta story) |

### Nota sobre `Capacitor.Plugins` vs imports ESM

`etograma.html` é um single-file sem bundler — não há `import`. Os plugins Capacitor 5+ são acessíveis via `window.Capacitor.Plugins` quando o WebView Android carrega o runtime. No browser normal, `Capacitor` é `undefined` — sempre verificar `backupCapacitorAtivo()` antes de acessar qualquer plugin.
