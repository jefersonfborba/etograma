## Story
**As a** usuário do Etograma no Android,
**I want** que o app solicite as permissões corretas e apareça com ícone e identidade visual adequados,
**so that** a experiência nativa seja profissional e todas as funcionalidades (microfone, storage) funcionem sem erros de permissão.

## Acceptance Criteria

1. O `AndroidManifest.xml` contém a permissão `RECORD_AUDIO` (necessária para `getUserMedia`)
2. O `AndroidManifest.xml` contém as permissões de storage adequadas para Android 9–14
3. O app aparece com ícone correto (192×192 adaptativo) na tela inicial do Android
4. O app aparece com nome "Etograma" no launcher do Android
5. Uma splash screen é configurada (cor sólida verde `#1a3c1f`, sem logo por ora)
6. O `targetSdkVersion` está em 34 (Android 14) e `minSdkVersion` em 24 (Android 7)
7. A orientação do app é forçada para `portrait` no `AndroidManifest.xml`
8. O build `npx cap sync` executa sem warnings de configuração

## Dev Notes

### Permissões por versão do Android

| Permissão | Android ≤ 9 | Android 10-12 | Android 13-14 |
|-----------|-------------|---------------|---------------|
| `WRITE_EXTERNAL_STORAGE` | Obrigatória | Limitada | Não existe |
| `READ_EXTERNAL_STORAGE` | Obrigatória | Obrigatória | Substituída por `READ_MEDIA_*` |
| `RECORD_AUDIO` | Obrigatória | Obrigatória | Obrigatória |

> Para exportação via `@capacitor/filesystem` com `Directory.Cache`, **não** são necessárias permissões de storage no Android 10+. As permissões de storage são relevantes apenas se o app precisar ler arquivos externos.

### Localização dos arquivos Android

```
android/
├── app/
│   ├── build.gradle                          ← SDK versions
│   └── src/main/
│       ├── AndroidManifest.xml               ← permissões, orientação
│       ├── res/
│       │   ├── mipmap-*/ic_launcher.png      ← ícones por densidade
│       │   └── values/
│       │       ├── strings.xml               ← app_name
│       │       └── styles.xml                ← splash screen
```

### Exemplo de `AndroidManifest.xml` após configuração

```xml
<manifest>
  <uses-permission android:name="android.permission.INTERNET" />
  <uses-permission android:name="android.permission.RECORD_AUDIO" />
  <uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"
    android:maxSdkVersion="32" />
  <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"
    android:maxSdkVersion="29" />

  <application
    android:icon="@mipmap/ic_launcher"
    android:label="@string/app_name">
    <activity
      android:screenOrientation="portrait"
      ...>
    </activity>
  </application>
</manifest>
```

### Geração de ícones

Usar ImageMagick (WSL) ou ferramenta online para redimensionar `icons/icon-512.png`:

```bash
# WSL/Linux
convert icons/icon-512.png -resize 48x48 android/app/src/main/res/mipmap-mdpi/ic_launcher.png
convert icons/icon-512.png -resize 72x72 android/app/src/main/res/mipmap-hdpi/ic_launcher.png
convert icons/icon-512.png -resize 96x96 android/app/src/main/res/mipmap-xhdpi/ic_launcher.png
convert icons/icon-512.png -resize 144x144 android/app/src/main/res/mipmap-xxhdpi/ic_launcher.png
convert icons/icon-512.png -resize 192x192 android/app/src/main/res/mipmap-xxxhdpi/ic_launcher.png
```

Alternativa: `npx @capacitor/assets generate` se `@capacitor/assets` for instalado.
