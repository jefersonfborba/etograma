## Story
**As a** desenvolvedor do Etograma,
**I want** configurar o Capacitor e reorganizar os arquivos do projeto em uma estrutura `www/`,
**so that** o app possa ser empacotado como um app Android nativo usando Capacitor como bridge.

## Acceptance Criteria

1. Node.js project inicializado (`package.json`) na raiz do projeto
2. Capacitor instalado (`@capacitor/core`, `@capacitor/cli`, `@capacitor/android`)
3. Arquivo `capacitor.config.ts` criado e configurado (`appId`, `appName`, `webDir: "www"`)
4. Pasta `www/` criada com os arquivos corretos:
   - `www/index.html` (conteúdo de `etograma.html`)
   - `www/sw.js`
   - `www/manifest.json`
   - `www/icons/icon-192.png`
   - `www/icons/icon-512.png`
5. Plataforma Android adicionada (`npx cap add android`) — pasta `android/` gerada
6. `npx cap sync` executa sem erros
7. Arquivos originais (`etograma.html`, `etograma-dev.html`, `etograma-test-1.png`, etc.) preservados na raiz (não deletados)
8. `.gitignore` atualizado para ignorar `node_modules/` e a pasta `android/` (exceto arquivos de configuração essenciais)

### `capacitor.config.ts` base

```typescript
import { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'br.com.etograma.app',
  appName: 'Etograma',
  webDir: 'www',
  server: {
    androidScheme: 'https'
  }
};

export default config;
```

> **Por quê `androidScheme: 'https'`:** O IndexedDB no Android WebView requer que o contexto seja `https://` (ou localhost). Sem isso, o storage pode ser bloqueado em algumas versões do Android.

### Atenção: Service Worker no contexto nativo

O `sw.js` existente funciona mas é redundante dentro do Capacitor — o bundle já é servido localmente. Não é preciso remover, apenas garantir que o path no `index.html` está correto (`./sw.js`, não `/sw.js`).

### Fonte da decisão arquitetural

Avaliação realizada por @architect em 2026-06-08: Capacitor selecionado sobre TWA e PWABuilder por suporte Android + iOS, controle total sobre APIs nativas e compatibilidade com IndexedDB.
