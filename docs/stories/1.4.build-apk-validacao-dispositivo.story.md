## Story
**As a** pesquisador de campo,
**I want** instalar o Etograma no meu Android via APK e usá-lo normalmente,
**so that** eu possa registrar comportamentos etológicos em campo sem depender de browser ou loja de aplicativos.

## Acceptance Criteria

1. O build APK é gerado com sucesso via Android Studio (ou `./gradlew assembleDebug`)
2. O APK pode ser instalado em um dispositivo Android físico via ADB (`adb install`) ou sideloading
3. O app abre e exibe a tela inicial corretamente após instalação
4. O app funciona 100% offline após instalado (sem conexão Wi-Fi)
5. Criação de projeto, adição de animais e registro de comportamentos funcionam normalmente
6. Os dados persistem entre fechamentos e reabertura do app (IndexedDB funcionando)
7. A vibração (`navigator.vibrate`) funciona no dispositivo
8. A exportação de CSV abre o share sheet nativo do Android com o arquivo correto
9. A rotação é bloqueada em portrait (app não gira ao inclinar o dispositivo)
10. Nenhum crash ou erro crítico no console do Android Studio durante uso normal

## Dev Notes

### Pré-requisitos para build

- Android Studio instalado (inclui SDK e Gradle)
- JDK 17+ configurado no PATH ou via `JAVA_HOME`
- ADB disponível: `adb --version` deve funcionar no terminal

### Alternativa: build via Android Studio (GUI)

1. Abrir Android Studio → "Open" → selecionar pasta `android/`
2. Aguardar sync do Gradle
3. Menu: Build → Build Bundle(s) / APK(s) → Build APK(s)
4. APK gerado em `android/app/build/outputs/apk/debug/`

### Instalar via sideloading (sem cabo)

1. Copiar `app-debug.apk` para o dispositivo (WhatsApp, Drive, email)
2. No Android: Configurações → Segurança → Instalar apps desconhecidos
3. Abrir o APK pelo gerenciador de arquivos → Instalar

### IndexedDB e migração de dados

O IndexedDB do app nativo é **isolado** do IndexedDB do browser. Dados criados no browser não aparecem no app nativo e vice-versa. Isso é esperado e não é um bug — é o comportamento padrão do WebView.

Se o usuário precisar migrar dados do browser para o app nativo, isso é escopo de uma story futura (migração de dados — PRD FR9).
